struct nv::merlin::EvictStrategy

The eviction strategies.

Note

The score is introduced to define the importance of each key, the larger, the more important, the less likely they will be evicted. On kLru mode, the scores parameter of the APIs should keep nullptr, the score for each key is assigned internally in LRU(Least Recently Used) policy. On kCustomized mode, the scores should be provided by caller.

Note

Eviction occurs automatically when a bucket is full. The keys with the minimum score value are evicted first.

Note

For kEpochLru and kEpochLfu, the high 32bits would be set to global_epoch while the low 32 bits is timestamp or frequency.

Note

on kLFU, when the scores reaches to the max of uint64_t, it will not increase any more.

Note

on kLkEpochLfu, when the lower 32bit stand for frequency reaches to the max of uint32_t, it will not increase any more.
