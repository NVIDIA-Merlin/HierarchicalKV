template<class Key, class V, class M, size_t DIM>
class nv::merlin::HashTable

Merlin HashTable is a concurrent and hierarchical HashTable powered by GPUs which can use both HBM, HMEM and SSD(WIP) as storage for Key-Values.

Todo:

:

  • Support dynamic rehashing

  • Support SSD/NVMe device as part of storage

Note

Supports concurrent upsert, but not concurrent upsert and find now. The upsert means insert or update if already exists.

Note

There is no API eviction and the eviction will happen automatically when table is almost full. We introduce a meta concept to help implement it. The keys with minimum meta will be evicted with priority. So the meta usually is the timestamp or times of the key occurrence. The user can also customize the meaning of the value that is equivalent to customizing the eviction policy.

Template Parameters

  • Key – type of the key

  • V – type of the Value’s item type, which should be basic types of C++/CUDA.

  • M – type of the meta and must be uint64_t in this release.

  • DIM – dimension of the vector

inline void insert_or_assign(const Key *keys, const V *vectors, size_t len, bool allow_duplicated_keys = true, cudaStream_t stream = 0)

Attempts to insert key-vectors pairs into the table. If one key already exists, the vector of the key will be updated.

Note

When the table is already full, the LRU policy will be applied: the key-vector pairs to be replaced is least recently inserted.

Parameters

  • keys – The keys to be inserted on GPU accessible memory.

  • vectors – The vectors to be inserted on GPU accessible memory.

  • len – Number of Key-Value pairs to be upsert.

  • allow_duplicated_keys – Flag of if allow the keys contains duplicate keys. If false, the caller should guarantee the keys has no duplicated keys, and the performance will be better.

  • stream – The CUDA stream used to execute the operation.

inline void insert_or_assign(const Key *keys, const V *vectors, const M *metas, size_t len, bool allow_duplicated_keys = true, cudaStream_t stream = 0)

Attempts to insert key-value-meta tuples into the table. If one key already exists, the value of the key will be updated.

Note

When the table is already full, the keys to be replaced are the keys with minimum meta value. If the meta of the new key is even less than minimum meta of the target bucket, the key will not be inserted.

Parameters

  • keys – The keys to be inserted on GPU accessible memory.

  • vectors – The vectors to be inserted on GPU accessible memory.

  • metas – The metas to be inserted on GPU accessible memory. The metas usually are with uint64_t value which could stand for the timestamp of the key inserted or the number of the key occurrences.

  • len – Number of Key-Value-Meta tuples to be upsert.

  • allow_duplicated_keys – Flag of if allow the keys contains duplicate keys. If false, the caller should guarantee the keys has no duplicate keys, and the performance will be better.

  • stream – The CUDA stream used to execute the operation.

inline void accum(const Key *keys, const V *vals_or_deltas, const bool *exists, size_t len, bool allow_duplicated_keys = true, cudaStream_t stream = 0)

Searches for each key in keys in the table. If the key is found and corresponding exist in exists is true, the vectors_or_deltas will be treated as delta against to the old value, and the delta will be add to the old value of the key. If the key is not found and corresponding exist in exists is false, the vectors_or_deltas will be treated as a new value and the key-value pair will be updated into the table directly.

Todo:

support accum with metas.

Note

Specially when the key is found and exist is false, or the key is not found and exist is true, nothing will be changed and this accum will be ignored, for we assume these situations occur while the key was modified or removed by other processes just now.

Parameters

  • keys – The keys to be inserted on GPU accessible memory.

  • vals_or_deltas – The vectors to be inserted on GPU accessible memory.

  • exists – if the key exists when last find in this process.

  • len – Number of Key-Value pairs to be processed.

  • allow_duplicated_keys – Flag of if allow the keys contains duplicate keys. If false, the caller should guarantee the keys has no duplicate keys, and the performance will be better.

  • stream – The CUDA stream used to execute the operation

inline void find(const Key *keys, V *vectors, bool *found, size_t len, const V *default_vectors, bool full_size_default, cudaStream_t stream = 0) const

Searches the table for the specified keys.

Note

When a key is missing, a default value in default_vectors will returned. Specially, if full_size_default is true, default_vectors will be treated as a V array with len * DIM, at this situation, each keys will have a different default value, or if full_size_default is true, the default_vectors only contains one default and each keys will share it when missed.

Parameters

  • keys – The keys to be searched on GPU accessible memory.

  • vectors – The vectors to be searched on GPU accessible memory.

  • found – The status indicates if the keys are found on GPU accessible memory.

  • len – Number of Key-Value pairs to be searched.

  • default_vectors – The default vectors for each keys on GPU accessible memory. If the keys are missing, the vectors in it will be returned.

  • full_size_default – true if the default_vectors contains the same size default vectors with keys.

  • stream – The CUDA stream used to execute the operation.

inline void reserve(size_type count, cudaStream_t stream = 0)

Sets the number of buckets to the number needed to accomodate at least count elements without exceeding maximum load factor and rehashes the table, i.e. puts the elements into appropriate buckets considering that total number of buckets has changed.

Note

If the count or double of the count is greater or equal than max_size_, the reserve will not happen.

Parameters

  • count – new capacity of the table.

  • stream – The CUDA stream used to execute the operation.
